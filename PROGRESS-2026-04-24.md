## 1. Purpose

Field measurement & quoting form used on tablet/phone by reps on-site at Anglo Windows. Captures per-unit type, dimensions, glass, colour, extras. Outputs to printable PDF. Persists to localStorage. Must also read back a handwritten rough copy (photo upload → extract data) and export to external systems (Google Sheet / DB / server).

Single-file HTML/CSS/JS. No framework. No external dependencies. Self-contained.

## 2. Business logic rules from `_6` that must be preserved

### Building type cascade (specs bar)
- `building_type = NEW` → shows Enclosure Type picker (NEW BUILDING / ENCLOSURE) + Wrap toggle. Animation pulses the enclosure field until selected.
- `building_type = FLAT / HOUSE / COMMERCIAL` → shows Outer Frame / Wall Type instead (not wrap).
- `enclosure_type = NEW BUILDING` → auto-sets `WRAP` on all rows.
- `enclosure_type = ENCLOSURE` → auto-sets `NO-WRAP` on all rows.
- Changing `wrap` manually → propagates the `Wrap` extra to every row.

### Glass rules
- Doors (category = DOOR) → `glass` forced to `GLASS-SAFETY`, `burgbar` forced to `NONE`. No override.
- `glass = GLASS-FROSTED` on spec bar → reveals a frosted qty input (compliance check — how many frosted units).

### Validation (`getMiss`)
- Missing fields flagged per row → red left border (`.rmiss`), green if complete (`.rok`).
- Builder rows skip sash-size validation (they carry `wbState` instead).

### Spec propagation & overrides
- Spec bar dropdowns (glass / burgbar / colour / HW colour) apply to all rows on change.
- "Apply to All" (⟲) button per spec group forces propagation.
- Per-row overrides possible; `toggleOverride()` flips them back.

### Door-specific
- Door stile: `60mm` / `90mm`. Default `60mm`. VALENCIA brand → no stile choice.
- Opening direction ("Open Inward") + lock position stored per row.
- PALACE vs VALENCIA distinction for multi-slide.

### Sash sizes
- Non-red presets: 600×600.
- Red block (W≥1800, H≥1200): 900×600.
- Legacy dropdown: STD / HALF / 315 / 545 / 845 / CUSTOM.

### Persistence
- localStorage key `aw_rc_v10`.
- Column widths also persisted.

## 3. What can be OCR'd from a hand-filled rough copy

From the example sheet provided (Nomonde Mbata, 10/04/26):

| Zone | Data | Confidence |
|------|------|------------|
| Top stamp row | Date (`10/04/26`), Job ref/number | High |
| Header block | Client name, address, phone | Medium (handwriting) |
| Rep stamp (top right) | Rep name (`Stephen Smith`), contact numbers, branch | High (printed stamp) |
| Specs row checkboxes | BRONZE, EX WOOD, FLAT, PLASTER, FC-BRICK, FLOOR # | High |
| Unit cells | Per cell: room label, W×H (e.g. `2500 × 1190`), small sketch → type inference | Dims high, sketch interpretation medium |
| Internal annotations | Sash sub-dimensions inside a sketch (e.g. `2100 / 900 / 860`) | Medium |

**Pipeline sketch:** photo → Anthropic vision API → structured JSON → populate `specs{}` + `rows[]`. Scaffold already exists in `_6` (`scan-bar` + AI extraction prompt). Needs hardening for the extra fields.

## 4. Ideas to import from Claude Design Standalone (noted, not built)

The Standalone is a React/JSX app — cannot merge directly. Port these design ideas as pure SVG strings into `wbRender()` and `wbMakeSVG()`:

- **FX cross symbol** — currently FIXED cells have no glyph. Dashed X makes them readable at-a-glance.
- **SHL / SHR split** — side hung left vs right. `_6` only has a single `SIDE` direction.
- **Hinge indicator rects** — small filled rects on the hinge edge of the sash.
- **Coloured glass fills** per type (light blue / green / etc.) vs current ghost tint.
- **Status badge system** — per-row pill: `Ready / Needs Size / Needs Type / Check Glass / Check Config`. Cleaner than the current validation tab.
